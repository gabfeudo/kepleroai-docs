## Prompt (Comportamento AI)

Un **prompt** è una breve istruzione o descrizione che definisce il comportamento, l'identità e la personalità del tuo assistente AI. In pratica, serve a spiegare all'AI **come deve rispondere** agli utenti, quali regole seguire e quale tono utilizzare nelle conversazioni.

Ad esempio, puoi usare il prompt per specificare se l'assistente deve essere formale o informale, se deve rispondere in modo sintetico o dettagliato, oppure se deve evitare certi argomenti.

<Note>
  Il prompt non è visibile agli utenti finali che parleranno con il tuo
  assistente AI: è uno strumento che ti permette di personalizzare l'esperienza
  e l'efficacia delle risposte dell'AI.
</Note>

## Best Practice per scrivere prompt efficaci

Scrivere un prompt efficace è fondamentale per ottenere risposte precise e coerenti dall'AI. Di seguito trovi le migliori pratiche basate sul framework **KERNEL**, un metodo testato che migliora significativamente la qualità dei risultati.

### K - Keep it simple (Mantienilo semplice)

La semplicità è la chiave per prompt efficaci. Evita descrizioni prolisse e concentrati su un obiettivo chiaro.

<Tabs>
<Tab title="❌ Non fare così">
```
Ciao, sono il proprietario di un negozio di abbigliamento online e vorrei
che tu mi aiutassi a scrivere qualcosa di interessante per i miei clienti
riguardo alla nuova collezione estiva che include vestiti, accessori e scarpe,
ma non so bene come strutturarlo e vorrei anche che fosse coinvolgente...
```
</Tab>

<Tab title="✅ Fai così">
```
Scrivi una descrizione promozionale di 100 parole per la nuova collezione
estiva di abbigliamento, con focus su vestiti leggeri e accessori colorati.
```
</Tab>
</Tabs>

<Tip>
Un prompt conciso riduce l'utilizzo di token del 70% e ottieni risposte 3 volte più veloci
</Tip>

### E - Easy to verify (Facile da verificare)

Il tuo prompt deve includere criteri di successo chiari e misurabili. Se non puoi verificare il risultato, l'AI non può offrirtelo con precisione.

<Tabs>
<Tab title="❌ Non fare così">
```
Scrivi una risposta coinvolgente per questo cliente
```
</Tab>

<Tab title="✅ Fai così">
```
Scrivi una risposta al cliente che:
- Sia lunga massimo 50 parole
- Includa 2 emoji pertinenti
- Usi un tono amichevole e professionale
- Contenga un invito all'azione chiaro
```
</Tab>
</Tabs>

<Note>
I prompt con criteri verificabili hanno un tasso di successo dell'85% rispetto al 41% dei prompt generici
</Note>

### R - Reproducible results (Risultati riproducibili)

Evita riferimenti temporali o contestuali vaghi che potrebbero cambiare nel tempo. Il prompt deve funzionare oggi come tra un mese.

<Tabs>
<Tab title="❌ Non fare così">
```
Rispondi seguendo le best practice attuali e le ultime tendenze
```
</Tab>

<Tab title="✅ Fai così">
```
Rispondi usando il tono formale standard del customer service,
con una lunghezza compresa tra 30 e 50 parole
```
</Tab>
</Tabs>

<Check>
I prompt riproducibili mantengono una coerenza del 94% su un periodo di 30 giorni
</Check>

### N - Narrow scope (Ambito ristretto)

Un prompt = un obiettivo. Non combinare richieste multiple in un'unica istruzione.

<Tabs>
<Tab title="❌ Non fare così">
```
Rispondi al cliente, salva i suoi dati nel CRM, crea un promemoria
per ricontattarlo domani e genera un report della conversazione
```
</Tab>

<Tab title="✅ Fai così">
**Prompt 1:**
```
Rispondi al cliente in modo professionale confermando la ricezione
della sua richiesta di assistenza
```

**Prompt 2:**
```
Salva questi dati del cliente nel CRM: nome, email, richiesta
```
</Tab>
</Tabs>

<Warning>
I prompt con obiettivo singolo hanno un tasso di soddisfazione dell'89% contro il 41% dei prompt multi-obiettivo
</Warning>

### E - Explicit constraints (Vincoli espliciti)

Specifica chiaramente cosa l'AI **non deve fare**, oltre a cosa deve fare. I vincoli riducono gli output indesiderati del 91%.

<Tabs>
<Tab title="❌ Non fare così">
```
Rispondi al cliente
```
</Tab>

<Tab title="✅ Fai così">
```
Rispondi al cliente seguendo questi vincoli:
- Non usare termini tecnici o gergali
- Non superare le 40 parole
- Non promettere sconti o offerte speciali
- Non chiedere dati sensibili come carte di credito
- Usa massimo 1 emoji
```
</Tab>
</Tabs>

### L - Logical structure (Struttura logica)

Struttura ogni prompt seguendo questo schema:

<Steps>
<Step title="Context (Contesto)" icon="info">
Fornisci il contesto necessario per comprendere la situazione.

```
Il cliente ha ricevuto un prodotto danneggiato e chiede un rimborso
```
</Step>

<Step title="Task (Compito)" icon="list-check">
Specifica cosa l'AI deve fare.

```
Scrivi una risposta empatica che offra una soluzione
```
</Step>

<Step title="Constraints (Vincoli)" icon="shield">
Definisci i limiti e le regole da rispettare.

```
- Massimo 50 parole
- Tono professionale ma caloroso
- Offri sostituzione gratuita del prodotto
- Non menzionare rimborsi in denaro
```
</Step>

<Step title="Format (Formato)" icon="file">
Indica il formato di output desiderato.

```
Output: Messaggio in italiano, formattato per WhatsApp
```
</Step>
</Steps>

### Esempio pratico completo

Vediamo la differenza tra un prompt generico e uno strutturato con il metodo KERNEL:

<Tabs>
<Tab title="❌ Prima (Generico)">
```
Aiutami a rispondere a questo cliente che si lamenta
```

**Risultato:** Risposta generica di 200 parole, non utilizzabile
</Tab>

<Tab title="✅ Dopo (KERNEL)">
```
CONTESTO: Cliente ha ordinato scarpe taglia 42 ma ha ricevuto taglia 40

COMPITO: Scrivi risposta di scuse e proponi soluzione

VINCOLI:
- Massimo 40 parole
- Tono empatico e professionale
- Proponi spedizione gratuita della taglia corretta
- Non menzionare rimborsi

FORMATO: Messaggio WhatsApp con 1 emoji
```

**Risultato:** Risposta precisa di 35 parole, perfettamente utilizzabile
</Tab>
</Tabs>

### Suggerimenti avanzati

<AccordionGroup>
<Accordion title="Concatena prompt semplici invece di crearne uno complesso">
  Invece di scrivere un prompt complicato che fa tutto, crea una sequenza di prompt semplici dove ognuno fa una cosa specifica. Questo approccio migliora la precisione e facilita il debug.

  **Esempio:**
  1. Primo prompt: Analizza il sentiment del messaggio del cliente
  2. Secondo prompt: Genera risposta appropriata in base al sentiment
  3. Terzo prompt: Ottimizza la risposta per WhatsApp
</Accordion>

<Accordion title="Testa i tuoi prompt con casi limite">
  Prima di applicare un prompt in produzione, testalo con situazioni estreme:
  - Richieste molto lunghe o molto corte
  - Clienti arrabbiati o molto educati
  - Richieste fuori dal normale ambito aziendale

  Questo ti permette di identificare debolezze nel prompt e migliorarlo preventivamente.
</Accordion>

<Accordion title="Monitora e itera">
  Dopo aver implementato un prompt:
  - Monitora le risposte generate dall'AI
  - Raccogli feedback dagli operatori
  - Identifica pattern di errori ricorrenti
  - Aggiorna il prompt con vincoli più precisi

  L'ottimizzazione dei prompt è un processo continuo.
</Accordion>
</AccordionGroup>
